## 问题记录

### 问题1: start命令在空目录中自动创建.long-run-agent目录 ✅ 已修复

- **场景**: 空项目
- **命令**: `lra start`
- **现象**: 在空目录中运行lra start时，会自动创建.long-run-agent目录但没有完全初始化，导致状态显示为"部分初始化"而不是"新项目"
- **影响**: 严重 - 混淆Agent对项目状态的判断，导致无法正确初始化项目
- **根本原因**: 
  - LRACLI.__init__中创建了LocksManager、BatchLockManager、RecordsManager、TemplateManager实例
  - TaskManager.__init__中创建了TemplateManager实例
  - 这些管理器在初始化时会自动创建目录（通过os.makedirs或SafeJson.write）
- **修复方案**: 
  - 将LRACLI中的所有管理器改为延迟初始化（lazy initialization）
  - 将TaskManager中的template_manager也改为延迟初始化
  - 使用@property装饰器实现按需创建
- **修复文件**: 
  - `long_run_agent/cli.py:96-120`
  - `long_run_agent/task_manager.py:16-27`

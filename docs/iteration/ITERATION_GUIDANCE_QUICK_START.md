# 迭代阶段引导机制 - 快速使用指南

## 🎯 核心概念

LRA v4.1 引入了**迭代阶段引导机制**，为 Agent 提供渐进式的优化路径。

**每个任务最多7次迭代，每次迭代有明确的目标和引导。**

## 🚀 提交任务

```bash
lra set task_001 completed
```

### 场景1: 正常进入下一迭代

```
🔍 自动运行质量检查...

❌ 质量检查未通过:
  • test: test_login_failed

进入优化循环 (1/7)

状态已自动更新为: optimizing
```

### 场景2: 提前完成

```
🔍 自动运行质量检查...

✅ 质量检查全部通过
   得分: 95/100

🎉 恭喜！任务可提前完成（迭代 3/7）

💡 你已完成所有必需的质量检查，可以选择：
   1. 提前完成（推荐）
   2. 继续优化（可选）

✅ 状态已自动更新为: truly_completed
   任务已提前完成！
```

### 场景3: 阶段卡住

```
🔍 自动运行质量检查...

⚠️ 警告：在【基础功能验证】阶段已尝试 3 次
   当前迭代: 1/7

❌ 质量检查仍未通过:
   • test

💡 建议选项：
   1. 强制进入下一阶段（放弃当前阶段目标）
      执行: lra set task_001 force_next_stage
   2. 继续尝试当前阶段
      继续工作并提交: lra set task_001 completed
```

## 📚 支持的模板

### 1. code-module (编程任务)

```
迭代1: 基础功能验证 → 修复测试失败、确保核心功能可运行
迭代2: 功能完善 → 补充测试用例、处理边界情况
迭代3: 代码质量提升 → 修复lint警告、改善代码结构
迭代4: 初步优化 → 性能瓶颈分析、关键路径优化
迭代5: 代码重构 → 消除重复代码、优化设计模式（有安全检查）
迭代6: 全面测试 → 集成测试、性能测试、边界测试
迭代7: 交付准备 → 完善文档、代码审查、最终交付
```

### 2. novel-chapter (写作任务)

```
迭代1: 完成初稿 → 完成主要情节、确保故事完整
迭代2: 情节完善 → 补充情节细节、完善转折
迭代3: 人物塑造 → 深化人物性格、增加对话
迭代4: 场景描写 → 增加环境描写、营造氛围
迭代5: 语言润色 → 优化表达、调整节奏
迭代6: 逻辑检查 → 检查情节逻辑、确认时间线
迭代7: 最终定稿 → 最终审读、格式整理
```

### 3. data-pipeline (数据分析)

```
迭代1: 数据可用 → 数据成功加载、基本流程跑通
迭代2: 数据质量 → 处理缺失值、异常值检测
迭代3: 功能完整 → 实现所有步骤、完善处理逻辑
迭代4: 性能优化 → 优化处理速度、减少内存占用
迭代5: 可视化 → 添加图表、完善可视化
迭代6: 文档编写 → 编写文档、添加注释
迭代7: 交付验证 → 完整测试、文档审核
```

### 4. doc-update (文档任务)

```
迭代1: 内容收集 → 收集文档素材、整理信息
迭代2: 结构设计 → 设计文档结构、规划章节
迭代3: 内容编写 → 编写核心内容、补充细节
迭代4: 格式优化 → 统一格式、美化排版
迭代5: 示例补充 → 添加示例、完善说明
迭代6: 审校修改 → 检查错误、优化表达
迭代7: 最终发布 → 最终审核、准备发布
```

### 5. task (通用任务)

```
迭代1: 理解与规划 → 理解目标、明确标准、制定计划
迭代2: 基础实现 → 完成核心、确保可用、验证正确
迭代3: 功能完善 → 补充遗漏、处理边界、提高完整
迭代4: 质量提升 → 优化实现、提高标准、消除问题
迭代5: 优化改进 → 性能优化、结构优化、细节完善
迭代6: 验证测试 → 全面测试、验证可靠、确认完整
迭代7: 交付准备 → 文档完善、最终检查、准备提交
```

## 💡 最佳实践

### 1. 遵循阶段引导

每个阶段都有明确的目标，建议按阶段工作，不要跳过。

### 2. 关注"本次重点"

迭代引导会明确告诉你本次应该做什么，专注于这些重点。

### 3. 适当忽略次要检查

某些阶段会提示"应该忽略"某些检查，这是正常的渐进式优化策略。

### 4. 利用提前完成

如果所有必需检查都通过了，不必等到第7次迭代，可以提前完成。

### 5. 阶段卡住时果断决策

如果某个阶段尝试3次仍未通过，可以考虑：
- 强制进入下一阶段（放弃当前阶段目标）
- 或寻求人工帮助

### 6. 重构前确保安全

代码重构阶段会提醒你：
- 确保测试覆盖率 ≥ 80%
- 本次不添加新功能
- 重构后立即运行测试

---

## ⚠️ 注意事项

1. **迭代次数限制**：每个任务最多7次迭代
2. **提前完成条件**：所有必需的质量检查必须通过
3. **阶段卡住阈值**：连续3次相同迭代视为卡住
4. **安全检查**：仅代码重构阶段有安全检查

### 2. TaskManager 扩展

**文件**: `lra/task_manager.py`

**新增方法** (5个):

1. **`get_iteration_stage(task_id, iteration=None)`**
   - 获取任务的当前迭代阶段配置
   - 从模板配置中加载阶段信息
   - 支持指定迭代次数查询

2. **`update_iteration_stage(task_id)`**
   - 更新任务的当前迭代阶段到 `ralph.current_stage`
   - 自动在 `increment_iteration` 时调用

3. **`get_stage_suggestion(task_id)`**
   - 获取格式化的阶段建议文本
   - 包含迭代进度、阶段名称、重点列表、详细建议

4. **`check_stage_stuck(task_id, threshold=3)`**
   - 检查任务是否在当前阶段卡住
   - 连续3次相同迭代视为卡住
   - 返回是否卡住和停留次数

5. **`can_complete_early(task_id)`**
   - 检查任务是否可以提前完成
   - 所有必需的质量检查通过即可
   - 不必走完7个阶段

**修改方法**:
- `increment_iteration()` - 自动更新迭代阶段
- `create()` - 初始化任务时添加 `current_stage` 字段

### 3. TemplateManager 扩展

**文件**: `lra/template_manager.py`

**新增方法** (4个):

1. **`load_iteration_stages(template_name)`**
   - 从模板 YAML 文件加载迭代阶段配置
   - 自动验证阶段配置有效性
   - 提供7阶段默认配置作为 fallback

2. **`_validate_stage(stage)`**
   - 验证阶段配置是否完整有效
   - 检查必需字段和类型
   - 验证迭代次数范围

3. **`_get_default_stages()`**
   - 返回默认的7阶段配置
   - 当模板未配置时使用

4. **`get_stage_by_iteration(template_name, iteration)`**
   - 获取指定迭代的阶段配置
   - 找不到时返回最后一个阶段

## 🎯 核心设计决策

### 1. 固定7阶段模型

**决策**: 所有任务都使用7个迭代阶段

**原因**:
- 清晰的渐进式优化路径
- 避免Agent迷失方向
- 符合人类工作习惯

**实现**:
- 每个模板定义自己的7个阶段
- 通用任务也有完整的思考阶段

---

### 2. 允许提前完成

**决策**: 所有必需检查通过即可提前完成

**逻辑**:
- 检查 `priority_checks` 是否全部通过
- 如果通过，允许提前退出（不必走完7阶段）
- 显示恭喜信息并自动标记为 `truly_completed`

**用户体验**:
- Agent看到"🎉 恭喜！任务可提前完成"
- 清楚知道已完成所有必需检查
- 可以选择继续优化或提前完成

---

### 3. 阶段卡住强制进入

**决策**: 同一阶段卡住3次后强制进入下一阶段

**逻辑**:
- 检查 `optimization_history` 最近3次迭代
- 如果 `iteration` 值相同，说明卡住
- 提供强制进入下一阶段的选项

**用户体验**:
- 显示警告："⚠️ 在当前阶段已尝试3次仍未通过"
- 提供2个选项：强制进入下一阶段 / 继续尝试
- 使用 `lra set <task_id> force_next_stage` 强制进入

---

### 4. 重构阶段安全检查

**决策**: 迭代5（代码重构）必须包含安全检查

**检查内容**:
- 测试覆盖率 ≥ 80%
- 功能锁定（本次不添加新功能）
- 回归测试（重构后必须运行全部测试）

**目的**:
- 避免重构导致功能崩溃
- 确保有足够的测试保护
- 提醒Agent重构风险

---

### 5. 只在 `lra show` 中显示详细引导

**决策**: 详细引导只在 `lra show` 中显示

**原因**:
- 避免 `lra context` 和 `lra set` 输出过于冗长
- Agent 主动查看详情时才显示完整引导
- 其他命令只做简略提示

**实现**:
- `lra show`: 完整的进度条、引导框、安全检查
- `lra context`: 只提示"任务正在迭代X/7"
- `lra set completed`: 只提示关键决策点

## 🎨 不同模板的迭代路径

### code-module (编程任务)

```
迭代1: 基础功能验证 → 让功能跑起来
迭代2: 功能完善 → 补充测试和边界情况
迭代3: 代码质量提升 → 修复lint、改善结构
迭代4: 初步优化 → 性能瓶颈分析
迭代5: 代码重构 → 消除重复、优化设计（有安全检查）
迭代6: 全面测试 → 集成测试、性能测试
迭代7: 交付准备 → 文档、审查、交付
```

### novel-chapter (写作任务)

```
迭代1: 完成初稿 → 完成主要情节
迭代2: 情节完善 → 补充细节、完善转折
迭代3: 人物塑造 → 深化性格、增加对话
迭代4: 场景描写 → 环境描写、营造氛围
迭代5: 语言润色 → 优化表达、调整节奏
迭代6: 逻辑检查 → 检查逻辑、确认时间线
迭代7: 最终定稿 → 审读、格式整理
```

### data-pipeline (数据分析)

```
迭代1: 数据可用 → 加载成功、流程跑通
迭代2: 数据质量 → 缺失值、异常值
迭代3: 功能完整 → 所有步骤、完整逻辑
迭代4: 性能优化 → 速度、内存
迭代5: 可视化 → 图表、美化
迭代6: 文档编写 → 文档、注释
迭代7: 交付验证 → 测试、审核
```

### doc-update (文档任务)

```
迭代1: 内容收集 → 收集素材、整理信息
迭代2: 结构设计 → 设计结构、规划章节
迭代3: 内容编写 → 核心内容、补充细节
迭代4: 格式优化 → 统一格式、美化排版
迭代5: 示例补充 → 添加示例、完善说明
迭代6: 审校修改 → 检查错误、优化表达
迭代7: 最终发布 → 审核发布
```

### task (通用任务)

```
迭代1: 理解与规划 → 理解目标、明确标准
迭代2: 基础实现 → 完成核心、确保可用
迭代3: 功能完善 → 补充遗漏、处理边界
迭代4: 质量提升 → 优化实现、提高标准
迭代5: 优化改进 → 性能优化、结构优化
迭代6: 验证测试 → 全面测试、验证可靠
迭代7: 交付准备 → 文档完善、最终检查
```
